# Микрофронтенд с PIXI.js игрой

## Описание

Во втором Project компоненте теперь загружается микрофронтенд с простой PIXI.js игрой.

## Структура проекта

```
trisemya/
├── src/                          # Основное приложение
│   ├── components/
│   │   ├── Projects.tsx         # Компонент со списком проектов
│   │   └── Project.tsx          # Обычный проект
│   └── types/
│       └── microfrontend.d.ts   # Типы для микрофронтенда
├── microfrontends/
│   └── pixi-game/               # Микрофронтенд с PIXI игрой
│       ├── src/
│       │   ├── PixiGame.tsx     # PIXI.js игра
│       │   └── main.tsx         # Входная точка
│       ├── package.json
│       └── vite.config.ts       # Настройки Module Federation
└── vite.config.ts               # Основная конфигурация
```

## Запуск

### Вариант 1: Запуск всего сразу (рекомендуется)

```bash
npm run dev:all
```

### Вариант 2: Запуск по отдельности

```bash
# Терминал 1 - микрофронтенд
npm run dev:microfrontend

# Терминал 2 - основное приложение
npm run dev
```

## Как это работает

1. **Микрофронтенд** (порт 3001) - содержит PIXI.js игру и экспортирует компонент через Module Federation
2. **Основное приложение** (порт 5173) - загружает микрофронтенд динамически во второй Project

## Особенности реализации

- **Module Federation**: Используется для загрузки микрофронтенда
- **Динамическая загрузка**: React.lazy() для асинхронной загрузки компонента
- **Fallback**: Показывается "Загрузка PIXI игры..." пока компонент загружается
- **Независимая разработка**: Микрофронтенд можно разрабатывать отдельно

## PIXI.js игра

Простая игра включает:

- Движущийся белый круг
- Анимация отскока от краев
- Интерактивность (клик меняет цвет)
- Текстовые подсказки

## Сборка для продакшена

```bash
npm run build:all
```

Это соберет сначала микрофронтенд, а затем основное приложение.

## Решение проблем

### Module Federation не работает?

Если вы видите ошибки типа "Failed to fetch remoteEntry.js", приложение автоматически переключится на iframe fallback. Это нормальное поведение.

### Проблемы с CORS

Убедитесь, что оба сервера запущены:

- Основное приложение: http://localhost:5173
- Микрофронтенд: http://localhost:3001

### Альтернативные подходы

1. **Module Federation** - идеальный вариант для продакшена
2. **iframe** - надежный fallback для разработки
3. **Обычный импорт** - для простых случаев

### Отладка

Откройте DevTools и проверьте:

- Консоль на ошибки загрузки
- Network tab для проверки запросов к remoteEntry.js
- Доступность http://localhost:3001/
